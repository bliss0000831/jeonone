## 1. 핵심 컨셉 — "광장"이란

### "광장 = 지역 단위 작은 도시 플랫폼"

다른 플랫폼이 "전국 1개 서비스" 라면, 광장은 **춘천광장·강릉광장처럼 도시·지역 단위로 독립된 미니 플랫폼이 모인 연방형 구조**입니다.

| 항목 | 일반 플랫폼 (당근·직방 등) | 광장 |
|---|---|---|
| 회원 단위 | 전국 통합, 지역 필터만 적용 | **광장 단위 회원** — 같은 동네 사람만 |
| 회원 정체성 | 닉네임·프로필 1개 (전국 공통) | **광장마다 다른 닉네임·프로필** — 강릉광장에선 다른 페르소나 |
| 운영 주체 | 본사 단일 운영 | **광장 관리자** — 지역 운영진이 자율 관리 |
| 서비스 폭 | 1 영역 (중고거래 only, 부동산 only) | **부동산 + 홈즈 + 거래 + 커뮤니티 + 생활 인프라 통합** |
| 지역 정체성 | 약함 | **강함** — "춘천 사람만의 광장" |

### 광장 격리 (Plaza Isolation)

기술적으로 모든 데이터는 `plaza_id` 컬럼으로 격리됩니다 (DB · API · UI 3중 차단):
- 강릉광장 admin 은 춘천광장 회원 절대 못 봄
- 강릉광장에서 글 등록 → 춘천광장에선 안 보임
- **단, 의도적 글로벌 영역**: 공동구매·로컬푸드는 "전체 광장 노출" 옵션 가능 (거래량 확보)

### 광장 허브 (Hub)

`/hub` 페이지: 전국의 모든 광장 목록. "오픈된 광장" / "오픈 예정 광장" 표시. 사용자는 광장을 골라 입장.

광장 입장하면 그 광장 전용 UI 로 진입 — 마치 다른 앱 같은 느낌.

```
        ┌─────────────────────────┴───────────────────────────┐

   ┌─────────┐                                          ┌──────────┐
```

## 3. 부동산 영역

### 3-1. 매물 (Property)

**한 줄** — 광장 내 매매·전세·월세·단기 매물 종합 마켓플레이스

#### 사용자 흐름

```
[등록자]
  사진 등록 → 주소 입력 → 지도 핀 자동 (geocode)
  → 매물 유형 · 거래 유형 · 가격 · 면적 · 옵션 입력
  → 360° 파노라마 사진 추가 (선택)
  → 동영상 추가 (선택)
  → 등록 완료 → 즉시 광장 매물 리스트 노출

[구매자]
  매물 리스트 (리스트 / 카드 / 지도 3가지 뷰)
  → 필터 (매물 유형 × 거래 × 가격 × 면적 × 지역 × 옵션)
  → 정렬 (최신 / 가격↑↓ / 면적 / 평당가 / 조회 / 찜)
  → 매물 상세 (사진 갤러리 + 정보 + 지도 + 360° + 판매자)
  → 전화 / 채팅 / 찜 / 공유 / 신고
```

#### 매물 유형
- 매물 유형: 아파트 / 빌라 / 오피스텔 / 단독 / 상가 / 토지 / 사무실
- 거래 유형: 매매 / 전세 / 월세 / 단기 / 전월세

#### 필터 옵션 (디테일)
- 매물 유형 (위 7종)
- 거래 유형 (위 5종)
- 판매자 유형: 전체 / 공인중개사 / 일반
- 가격 범위 (최소 ~ 최대)
- 면적 범위 (최소 ~ 최대 m²)
- 지역 (시/군 단위 다중 선택)
- 옵션: 풀옵션 / 주차가능 / 반려동물 / 엘리베이터 등

#### 지도 모드
- Naver Maps Native SDK 사용
- 광장 중심 좌표로 워밍업 (앱 시작 시 타일 캐시 prewarm)
- 클러스터링 + 가격 라벨 마커
- 마커 탭 → InfoWindow 카드 (사진+가격+상세보기)
- 위성/일반 토글
- 가까운 매물 우선 표시 (선택 시 GPS)

#### 빠른 필터 칩
- 내 주변 / 인기매물 / 신규

#### 정렬 옵션
- 최신순 (effective_at — 끌어올린 글 우선)
- 가격 낮은순 / 높은순
- 면적 넓은순 / 좁은순
- 평당가 낮은순
- 조회순 / 찜많은순

#### 판매자 정보
- 일반 / 공인중개사 구분 뱃지
- 공인중개사: 인증 마크 + 사무소 정보
- 가입 일수 표시 ("가입 N일차")

### 3-2. 구해주세요 (Property Requests)

**한 줄** — 매물이 아니라 **"이런 매물 구합니다"** 라는 역방향 요청 시스템

#### 사용자 흐름
```
[구매 희망자] "동내면 25평 매매 5억 이내 구합니다"
              → 글 등록 (이미지 없는 텍스트 위주)
              → 광장 노출

[중개사·매물 보유자] 요청 글 조회
                  → 매칭 매물 댓글 등록 (사진+가격+주소)
                  → 구해주세요 작성자에게 알림
                  → 채팅 연결
```

#### 상태 관리
- 모집중 (open) — 초록 뱃지
- 매칭됨 (matched) — 파랑 뱃지
- 마감 (closed) — 회색 뱃지

#### 의도
일방향(매물 등록 → 구매자 검색) 시장에서 **양방향 매칭** 가능. 매물이 흔한 지역엔 매매자가 활발, 매물 귀한 지역엔 구매자가 활발. 광장은 양쪽 다 지원.

## 4. 홈즈 — 생활 서비스

광장 내 전문가 서비스 4종 (홈즈 = 홈 서비스). 각 서비스가 독립된 마켓플레이스.

### 4-1. 인테리어 (Interior)

#### 사업자 흐름
```
[인테리어 업체]
  사업장 등록 → 시공 포트폴리오 사진 (전후 비교 가능)
  → 전문 분야 (아파트 / 빌라 / 상가 / 부분 시공)
  → 서비스 영역 (시/군 다중 선택)
  → 평당 단가 · 영업 시간 · 연락처
  → 등록 완료 → 광장 인테리어 카테고리에 노출
```

#### 사용자 흐름
```
[견적 받고 싶은 사용자]
  인테리어 리스트 → 포트폴리오 보기 → 평점·후기 확인
  → 채팅으로 견적 문의 → 방문 일정 조율
  → 시공 완료 후 후기 작성
```

#### 핵심 기능
- 사업자 등록증 인증 시 ✅ 뱃지
- 시공 후 사용자 후기 시스템 (별점 + 텍스트 + 사진)
- 평균 평점 · 시공 건수 표시
- "이번 주 견적 요청 N건" 등 활동 지표

### 4-2. 이사 (Moving)
- 이사 유형 (포장·반포장·일반)
- 차량 톤 수
- 평형별 견적 (1.5톤·2.5톤·5톤)
- 비수기/성수기 가격 차등 표시
- 작업 가능 거리

### 4-3. 청소 (Cleaning)
- 청소 유형 (입주·이사·정기·특수)
- 평수별 견적
- 청소 도구 보유 여부
- 친환경 인증

### 4-4. 수리 (Repair)
- 수리 카테고리 (전기·설비·도배·바닥·창호·기타)
- 출장비 / 시간당 단가
- 응급 수리 가능 여부

### 4-5. 홈즈 공통 기능

#### 후기 시스템
- 시공 완료 후 의뢰자가 후기 작성 (사진 첨부 가능)
- 별점 1~5 + 텍스트 + 시공 결과 사진
- 업체는 사장님 답글 가능
- 거짓 후기 신고 가능

#### 매칭 방식
1. **검색 → 직접 연락** — 사용자가 업체 골라 채팅
2. **구해주세요 → 견적 응답** — 사용자가 요청 → 업체들이 견적
3. **초대장 → 전문가 매칭** — 사용자 글에 전문가 자동 추천 (아래 8-1 참조)

## 5. 지역 거래 영역

### 5-1. 중고거래 (Secondhand)

#### 흐름
```
[판매자]
  사진 (여러 장) → 제목 · 가격 · 설명
  → 동(洞) 단위 위치 → 카테고리
  → 등록 → 광장 노출

[구매자]
  중고거래 리스트 → 필터 (카테고리·가격·동) · 검색
  → 상세 → 찜 / 채팅 / 신고
  → 거래 약속 → 만나서 직거래
```

#### 핵심 기능
- 카테고리: 디지털·가전·가구·생활/주방·의류·뷰티·도서·취미·스포츠·반려·기타
- 가격 입력 (나눔도 가능 = 0원)
- 사진 최대 10장
- 동(洞) 단위 위치 표시 ("거두1동" 식)
- 끌어올리기 (Bump) — 24시간마다 1회 무료 + 추가는 결제
- 찜 / 숨김 / 신고
- 거래 완료 표시 (판매자가 수동)
- 채팅 시 안전거래 안내 (아직 미도입, Phase 2 예정)

#### 거래 상태
- 판매중 (active)
- 예약중 (reserved) — 거래 약속 잡힌 상태
- 거래완료 (sold)
- 숨김 (hidden) — 신고 누적 3회 자동
- 삭제 (deleted)

### 5-2. 나눔 (Sharing)

**한 줄** — 무료 나눔 전용. 가격 입력 없음.

#### 흐름
- 사진 + 제목 + 설명 + 위치 → 등록
- "나눔 신청" 채팅 → 픽업 약속
- 받은 사람은 인증샷 업로드 (선택) → 광장 미담 코너

#### 운영 정책
- **영구 무료** — 수익화 절대 안 함
- 거래 분쟁 시 광장 관리자 중재
- 1인당 월 나눔 횟수 제한 없음 (제한 없을 때 가치 발생)

### 5-3. 공동구매 (Group Buying)

#### 흐름
```
[판매자 (생산자·중소기업·동네 가게)]
  공구 글 등록:
    상품 사진 · 정가 · 공구가 · 모집 인원 · 마감일
    배송/픽업 방식 · 결제 방식
  → 광장 노출 (광장만 vs 전체 광장 선택)
  → 모집 진행 (실시간 참여자 수 표시)

[구매자]
  공구 리스트 → 상세 → "참여하기"
  → 결제 (PortOne/Toss — Phase 2)
  → 모집 마감 시:
    · 인원 채워짐 → 출고/배송
    · 인원 미달 → 자동 환불
```

#### 핵심 기능
- 모집 인원 · 마감일 · 정가/공구가 표시
- 실시간 참여자 수 (현재 N / 목표 M)
- 카테고리: 식품·생활·뷰티·디지털·반려·기타
- 배송 옵션: 택배 / 직접픽업 / 둘 다
- **광장 노출 옵션**: 자기 광장만 vs 전체 광장 (cross-plaza)
- 판매자 채팅으로 1:1 문의
- 그룹 채팅방 자동 생성 (참여자 전체 + 판매자)

#### 의도
지역 중소 생산자가 "10명 모이면 출고" 식으로 판매 가능. 대형 유통 마진 없이 직거래.

### 5-4. 로컬푸드 (Local Food)

**한 줄** — 지역 농산물·식품 직거래

#### 차이점 (vs 공동구매)
- 공동구매: 인원 모집 후 출고 (모집 마감 있음)
- 로컬푸드: 상시 판매 (재고만 있으면 즉시 결제)

#### 카테고리
- 농산물 (과일·채소·곡물)
- 축산 (한우·돼지·닭·계란)
- 수산 (어류·해산물)
- 가공식품 (장·꿀·반찬·김치)
- 음료 (전통주·차)
- 기타

#### 핵심 기능
- 재고 수량 표시 + 자동 카운트
- 산지 표시 (○○면 ○○농장)
- 농가 인증 뱃지 (조합 가입 시)

- 정기 구독 옵션 (매주 · 격주 · 월간)

### 5-5. 신장개업 (New Store)

#### 흐름

- 오픈 첫 주 할인 쿠폰 등록 가능 (광장 한정 발급)

## 8. 전문가 매칭 — 초대 · 구해주세요

### 8-1. 전문가 초대 시스템

**한 줄** — 일반 사용자가 채팅에서 "전문가 좀 추천해주세요" 하면, 광장의 전문가들 (공인중개사·인테리어·이사·청소·수리 5종) 에게 동시 초대장이 발송되어 빠르게 견적 모임.

#### 흐름
```
[일반 사용자]
  채팅방 (예: 누군가와 인테리어 상담 중)
  → "+ 전문가 초대" FAB 탭
  → 카테고리 선택 (인테리어·청소·수리·이사·중개사)
  → 인근 전문가 자동 매칭
  → 동시에 N명에게 초대장 발송

[전문가 (인테리어 업체 등)]
  알림 받음 → /invitations 진입
  → 대기/처리 섹션에 초대 표시 (어떤 사용자가 어떤 카테고리)
  → 수락 / 거절
  → 수락 시 채팅방 자동 입장 → 견적 제시
```

#### 의도
사용자는 일일이 업체 찾을 필요 X. **전문가 풀에 한 번에 요청 → 빠른 견적 비교**.

### 8-2. 매물 구해주세요 (앞 3-2 참조)

매물 영역의 역방향 매칭 시스템.

### 8-3. 신청 → 응답 (Requests)

```
[/requests/new]  — 새 요청 작성 (구해주세요)
[/requests]      — 요청 리스트
[/requests/[id]] — 상세 + 응답 (매물 추천)
```

응답한 매물은 신청자 알림으로 전달. 채팅 연결.

## 9. 채팅 · 알림 · 결제

### 9-1. 채팅 (Chat)

#### 채팅방 유형
1. **1:1 채팅** — 매물·중고·서비스 등 모든 거래의 기본
2. **그룹 채팅** — 모임원 · 공구 참여자
3. **광장 간 채팅** — 공구·로컬푸드 cross-plaza 거래 시 (강릉 ↔ 춘천)
4. **클럽 채팅** — 모임 전용 멤버 채팅방

#### 기능
- 실시간 메시지 (Supabase Realtime)
- 사진 / 파일 / 위치 공유
- 핀 카드 (거래 중인 매물·상품을 채팅방 상단 고정)
- 타이핑 인디케이터 ("상대가 입력 중")
- 읽음 표시
- 사용자 차단 (BlockedManagerSheet)
- 방 나가기 / 신고
- 채팅방 내 "+ 전문가 초대" FAB
- 메시지 검색
- 안 읽은 메시지 수 (탭 뱃지)

#### Cross-Plaza 채팅
공구·로컬푸드의 cross-plaza 거래에선 양 광장 사용자가 한 채팅방에 모임. 각자의 광장 닉네임으로 표시.

### 9-2. 알림 (Notifications)

#### 종류
- 채팅 메시지
- 좋아요 · 댓글
- 매물 · 중고 거래 진행 알림
- 모임 가입 승인 · 정기 모임
- 공구 모집 마감 · 출고
- 후기 받음
- 전문가 초대 수락/거절
- 신고 처리 결과
- 공지사항

#### 채널
- 인앱 알림 (notifications 화면)
- 푸시 알림 (Expo Notifications)
- 이메일 (선택)

#### 설정 (마이페이지 → 설정)
- 채널별 ON/OFF
- 야간 무음 시간
- 푸시 토큰 자동 관리

### 9-3. 결제 (Payments)

#### 통합 결제 인프라
- PG: **PortOne (구 아임포트)** 또는 **Toss Payments**
- 결제 수단: 카드 / 계좌이체 / 휴대폰 / 카카오페이 / 네이버페이
- 빌링키 저장 (자동 갱신 구독용)

#### 결제 시점
1. **공구 참여** — 즉시 결제 (모집 마감 미달 시 자동 환불)
2. **로컬푸드 주문** — 즉시 결제
3. **구독료 (월정액)** — 자동 갱신 (빌링키)
4. **올리기권 (Bump)** — 일회성 충전
5. **AI 크레딧** — 동영상 AI 생성 크레딧 충전

#### 구독 결제 흐름
```
사용자 가입 → 6개월 무료 기간 → 6개월 후
  → 자동 결제 시도 (저장된 빌링키)
  → 실패 시 5일 grace period → 카드 등록 안내
  → 미해결 시 구독 정지 (서비스 일부 제한)
```

#### 환불 정책
- 공구 모집 미달 → 즉시 자동 환불 (전액)
- 사업자 구독료 → 일할 환불 가능
- 사용한 올리기권·크레딧 → 환불 불가
- 디지털 상품 (크레딧) — 7일 내 미사용 시 환불

### 10-2. 마이페이지 메뉴

```
[MY 탭]
  ├── 프로필 (광장별 닉네임·아바타·자기소개)

  │     ├── 적립 (글 작성·후기·추천)
  │     └── 사용 (할인·교환)
  ├── 내 구독 (subscription) — 활성 구독 + 다음 결제일
  ├── AI 크레딧 (credits) — 동영상 AI 생성용
  ├── 정산 (settlement) — 판매자 정산 내역
  ├── 내가 쓴 글 (posts) — 모든 도메인 통합
  ├── 판매 관리 (sales) — 거래 상태별
  ├── 구매 내역 (orders)
  ├── 내 매물 (properties)
  ├── 찜 목록 (favorites)
  ├── 최근 본 글 (recent)
  ├── 후기 (reviews) — 받은 후기 + 작성한 후기
  ├── 하이라이트 (highlights) — 인스타 스토리 같은 24h 게시물
  ├── 동영상 (videos) — AI 생성 영상 모음
  ├── 팔로워 (followers) — 팔로우/팔로잉
  ├── 차단 목록 (blocked)
  └── 설정 (settings)
        ├── 알림 채널별 ON/OFF
        ├── 다크 모드 (light · dark)
        ├── 언어
        ├── 광장 변경 (다른 광장 입장)
        ├── 로그아웃
        └── 회원 탈퇴
```

## 12. 기술 / 운영 기반

### 12-1. 모바일

- React Native + Expo SDK 54
- Expo Router (file-based routing)
- iOS · Android 모두 지원
- 다크 모드 (light · dark 2-state)
- 푸시 알림 (Expo Notifications)
- OTA 업데이트 (EAS Update) — 코드 변경 시 앱 재설치 없이 즉시 배포

### 12-2. 웹

- Next.js 14 (App Router)
- SSR + RSC
- Vercel 자동 배포
- 반응형 (모바일·태블릿·PC)
- PC 에서는 관리자 페이지 + 부동산 매물 전체 보기 강점

### 12-3. 백엔드

- Supabase (PostgreSQL 15)
- Row Level Security (RLS) — DB 차원 권한
- Realtime — 채팅 · 알림 즉시 반영
- Edge Functions — Serverless 백엔드 로직
- Storage — 사진·동영상 저장
- 자체 CDN + 정적 지도 캐시 (s-maxage 30일)

### 12-4. 외부 연동

- 네이버 지도 (Mobile Native SDK + Static Maps API)
- 카카오 OAuth (로그인)
- 다음 우편번호 (주소 검색)
- 네이버 Geocode (좌표 → 주소)
- 한국 정부 공공 데이터 (주유소·화장실)
- PortOne / Toss Payments (결제)
- KT SMS (휴대폰 인증)

### 12-5. 광장 격리 (Plaza Isolation) — 3중 차단

1. **DB 차원** — RLS 정책으로 `plaza_id` 안 맞는 row 못 읽음
2. **API 차원** — 모든 admin API 가 현재 광장 컨텍스트 검증
3. **UI 차원** — 화면 쿼리에 `eq("plaza_id", plaza)` 명시
4. + **STRICT overlay** — plaza_profiles 행 없으면 사용자 자체가 안 보임 (다른 광장 닉네임 누출 방지)

### 12-6. 안전성

- 신고 시스템 (3회 자동 숨김)
- 키워드 자동 차단
- 사용자 차단 · 정지 (광장 admin 권한)
- 감사 로그 (모든 admin 행위 불변 기록)
- 광장 간 정보 격리 (RLS + 응용 + UI 3중)

## 광장이 만드는 것 — 우리가 바라는 그림

```
[중앙집중형 거대 플랫폼]              [광장]
당근 · 직방 · 네이버 — 본사가         각 동네 광장 관리자가 자율적으로
모든 정책 결정, 알고리즘이 추천         이웃의 데이터를 신뢰 기반으로 관리

전국 1억명 중에 매칭                  광장당 1만 ~ 10만명 중에 매칭
                                       — 진짜 우리 동네 사람만

광고 매출 → 본사로                    수익 60~80% 광장 관리자로
                                       — 광장이 자기 동네에 재투자
```

광장은 **연방형 지역 플랫폼** 입니다. 본사는 결제·기술·인프라만 책임지고, **각 광장이 자기 동네 색깔로 운영** 합니다.
