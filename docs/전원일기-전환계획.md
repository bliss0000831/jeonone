# 전원일기 전환 계획서

> **광장(Gwangjang)** 코드베이스를 **전원일기** — 농업인을 위한 농기구 직거래·대여·경매 + 로컬푸드 + 마을 커뮤니티 플랫폼으로 전환.
> **도 단위 멀티테넌시**: "강원 전원일기 / 전북 전원일기 …" 처럼 도별 독립 운영.
> 목표 디자인: 시안(`gwjeonwon.vercel.app`) 기준. 앱·웹 동일 UI. (시안은 별도 프로토타입 — 이 광장 코드를 그 디자인으로 전환)

## 0. 확정 방향

| 항목 | 결정 |
|---|---|
| 브랜드 | **{도}+전원일기** (예: 강원 전원일기) / 슬로건 "{도} 농업인을 위한 따뜻한 마을 장터" / 그린 테마 |
| 멀티테넌시 | 기존 **plaza 격리 시스템 유지**. 단위를 시군 → **도 단위 9개**로 재편 |
| 지역 단위 | 광장 = 도(강원/전북…), 시군(강릉시 등)은 **sub-region 필터**로 |
| 기능 | 기능을 **버리지 않고 농업용으로 재구성**. 핵심: 농기구 직거래·대여·경매 |
| DB | 새 Supabase에 **기존 스키마 재사용 + 농기구/대여/경매 확장** |
| 홈 화면 | 부동산 중심 → **농업 커뮤니티 허브형**으로 전면 재설계 |
| 경매 | 방식 미정 → 스키마 뼈대만, 실시간/기간제 양쪽 수용 |

## 멀티테넌시 (지역별 전원일기) 구조 — 유지

> 기존 광장의 plaza 격리를 그대로 활용. 글은 `plaza_id`로 도별 격리, 도별 독립 운영.

- **`plazas` 테이블**: `id`(서브도메인=도) · `name`(강원전원일기) · `parent_region` · 좌표/경계 · `theme` JSONB(색상·로고) · `is_active`/`is_open_soon`
- **브랜딩**: `site_labels`의 `{{plaza_city}}` 토큰 치환 → 도명 자동 반영. 도별 색·로고는 `plazas.theme`
- **서브도메인**: `gangwon.전원일기도메인` / `jeonbuk.…` — `lib/plaza/config.ts`의 `KNOWN_PLAZAS`
- **운영**: 서브관리자(`plaza-admin`)가 도별 공지·게시물 관리, 슈퍼관리자가 전체

### 광장 재편 (시군 60개 → 도 단위 9개)
`gangwon` · `gyeonggi` · `chungbuk` · `chungnam` · `jeonbuk` · `jeonnam` · `gyeongbuk` · `gyeongnam` · `jeju`
(서울·인천 등 광역시는 운영정책에 따라 추가 결정)
- `KNOWN_PLAZAS` / `ACTIVE_PLAZAS` 재정의
- `plazas` 테이블 시드 도 단위로 교체, 시군 좌표는 `sub-region-coords.ts`로 이동
- 위치 선택 UI: 도(광장) 진입 후 시군은 in-app sub-region 필터

## 권장 진행 순서

```
Phase 0 (DB) → 1 (브랜딩·지역) → 2 (홈 재설계) → 3 (기능 정리)
   → 4 (농기구 직거래) → 5 (대여) → 6 (경매) → 7 (농업 위젯) → 8 (QA·배포)
```

- **MVP 라인 (Phase 0~4 + 7 일부)**: 브랜딩 + 농업 허브 홈 + 농기구 직거래 + 로컬푸드 + 커뮤니티 + 날씨/공지 → 우선 런칭 가능
- 대여·경매는 모듈로 얹어 단계적 출시
